# antiX Customization — Known Errors & Fixes

| Error | Cause | Fix | Date |
|-------|-------|-----|------|
| zram not active after reboot | antiX-26 runit does NOT run `/etc/rc.local` — it runs `/etc/runit/rc.local`. Even that is fragile: the rc-local service uses `set -e` which silently kills scripts on any error | Create dedicated runit service at `/etc/sv/zram/run` with explicit `PATH=/sbin:/usr/sbin:/bin:/usr/bin`, wait loop for `/dev/zram0`, and `exec sv pause` at end. Symlink to `/etc/runit/runsvdir/default/` | 2026-04-05 |
| tint2 17.0.1 segfault with `strut_policy = none` | glib slice allocator bug in tint2 17.0.1 on Debian Trixie | Launch tint2 with `G_SLICE=always-malloc` environment variable. tint2 itself prints this hint: "Run tint2 with G_SLICE=always-malloc in case of strange behavior or crashes" | 2026-04-05 |
| tint2 dock autohide causes window resize | `doNotCover: 1` in `~/.icewm/winoptions` tells IceWM to never let windows cover the dock → IceWM resizes windows to make room. Combined with any `strut_policy` other than `none`, windows constantly resize as dock shows/hides | Remove `tint2.Tint2.doNotCover` from winoptions AND set `strut_policy = none` in tint2 config. Restart IceWM after winoptions change | 2026-04-05 |
| tint2 autohide unreliable — dock doesn't appear on mouse hover | Multiple causes: (1) `autohide_show_timeout > 0` adds delay perceived as broken, (2) `panel_layer != top` allows maximized windows to cover trigger strip, (3) IceWM edge switching steals mouse events | Set `autohide_show_timeout = 0.0`, `panel_layer = top`, `panel_dock = 0`, `wm_menu = 0`. Disable IceWM edge switching: `VerticalEdgeSwitch=0`, `HorizontalEdgeSwitch=0`, `EdgeSwitch=0` in prefoverride | 2026-04-05 |
| tint2 dock appears left-aligned instead of centered | `panel_size = 0 48` means full-width panel, icons start at left edge | Use `panel_size = 50% 48` with `panel_shrink = 1` — shrinks panel to icon width and centers it | 2026-04-05 |
| zstd zram on Atom x5-Z8350 causes CPU overhead | zstd is 3-5x slower than lz4 on Cherry Trail Atoms (no AVX2 support) | Use `lz4` algorithm, not `zstd`. lz4 is already default in antiX kernel. 2.63x compression vs 3.37x but negligible CPU cost | 2026-04-05 |
| swappiness=60 causes unnecessary swap on idle | Kernel pushes pages to zram even with 1.3GB free RAM, kswapd0 visible in top | Lower to 30 if idle CPU matters. 60 is safe but aggressive for Atom. For desktop use 30-40 is better balance | 2026-04-05 |
